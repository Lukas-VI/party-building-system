# 发展党员全程记实表整理与研发参考

## 1. 当前定位
《发展党员全程记实表》当前作为本项目的研发参考文档使用，主要作用是为 `25 步流程` 中“每一步需要填写什么、提交什么、由谁确认什么”提供结构化依据。

本阶段应明确：
- 该表当前 **不是** 已落地的电子档案模块说明
- 该表当前 **不是** 最终字段映射结果
- 该表当前 **不是** 直接入库的数据模型定义

本阶段的正确用途是：
- 作为后续 `25 步 -> 字段/材料/签署项` 梳理的参考底稿
- 作为会议纪要、纸质表格、后续需求访谈之间的对照文档
- 作为后续服务号网页 App 与 PC 后台表单设计的依据之一

## 2. 表单来源
- 制表单位：中共河南省委组织部
- 当前整理依据：
  - 原始扫描件 `发展党员全程记实表.pdf`
  - 人工整理出的字段结构
  - 现有会议纪要与讨论记录

说明：
- 当前 PDF 为扫描件，不能作为稳定的自动抽取来源
- 本文档采用人工整理结果，仅用于研发阶段结构对照
- 后续若会议纪要、业务口径与纸质表格存在冲突，以进一步确认后的需求为准

## 3. 表内使用说明
1. 入党申请人递交入党申请书后，即由所在党支部填写本表。在确定入党积极分子、确定发展对象、接收预备党员和预备党员转正等环节，党支部向上级党委报送备案材料和审批材料时，应一并将本表交上级党委审核签字，党委反馈意见后再退还党支部。
2. 正式纸质填写须使用黑色或蓝黑色墨水，日期统一使用公历和阿拉伯数字。无内容栏目应标注“无”，填写不下时可加附页。
3. 在上级党组织批准预备党员转为正式党员后，应及时将本表存入本人档案。

## 5. 研发中的使用原则
- 当前应把该表理解为“内容来源”，而不是“实现结果”
- 在未完成进一步需求确认前，不应直接把全部字段硬编码进页面或数据库
- 每个阶段先梳理：
  - 哪些是申请人填写
  - 哪些是审核者填写
  - 哪些是会议讨论/公示/审批结论
  - 哪些是材料上传项
  - 哪些只需要日志留痕
- 列表项如“本人简历”“培养联系人”“入党介绍人”“审查人”等，当前只整理结构，不在本轮决定最终前端交互和数据库形态

## 6. 与 25 步流程的关系
- `25 步流程` 仍是当前系统的正式办理主线
- 《发展党员全程记实表》当前作为 25 步中“字段、材料、意见、签名、时间点”整理的参考依据
- 不能简单理解为“一步对应一页表”
- 也不能在没有进一步会议纪要确认的情况下，直接把本表字段等同于最终系统字段

当前推荐的关系是：
- 先按 25 步梳理每步需要提交/确认的内容
- 再对照本表补齐阶段性字段
- 后续如确认需要正式电子档案模块，再单独设计数据模型与导出样式
