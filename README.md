# 党员发展管理系统

基于微信服务号网页 App、Node.js 服务端和 PC 后台的党务管理项目，覆盖发展党员流程、角色权限、流程审核、统计分析、材料上传和消息提醒。仓库中仍保留历史小程序实现，但已退出当前主建设链路。

## 项目介绍
- 服务号网页 App：申请人、支部审核者、组织员和学院党委共用的微信内轻量工作台。
- 服务端：统一认证、权限、流程、统计、导出、附件上传。
- PC 后台：查看台账、审核审批、统计分析、流程配置、组织与角色管理。
- 历史小程序：保留为旧实现存档，不再作为当前验收口径。

Gitee 仓库地址：
- [https://gitee.com/Yanhe06/party-building-mini-app.git](https://gitee.com/Yanhe06/party-building-mini-app.git)

## 业务场景
- 发展党员 25 步流程留痕管理
- 申请人基础资料维护
- 支部/单位/组织部多级权限操作
- 注册审核、流程审核、统计下载

## 角色与权限
- 入党申请人：查看与提交本人流程和资料
- 党支部书记：查看本支部、处理支部审核节点、导出本支部数据
- 组织员：查看本单位、审核注册、推进流程、分配角色
- 二级单位党委书记/副书记：查看与审批本单位范围内事项
- 校党委组织部：全校查看、统计、审核、导出
- 超级管理员：组织配置、角色配置、流程配置、全量导出

## 本地开发

### 1. 启动服务端
```bash
cd server
npm install
cp .env.example .env
npm run start
```

说明：
- 服务端默认连接 MySQL
- 启动时会执行 `server/deploy/mysql-init.sql` 并初始化演示数据
- 开发联调配置请参考独立调试文档

### 2. 启动后台
```bash
cd admin-web
npm install
npm run dev
```

可通过 `admin-web/.env.example` 配置 `VITE_API_BASE`。
如需按开发服务器联调口径启动后台前端：
```bash
npm run dev:1919
```
如需通过统一入口 `https://域名/web-admin/` 暴露后台：
- 桌面后台实际挂载在 `/web-admin/desktop/`
- 服务号网页 App 实际挂载在 `/wx-app/`
- 统一网关会根据设备类型把 `/web-admin/` 自动分流到桌面版或服务号网页端

### 3. 启动服务号网页 App
```bash
cd admin-mobile
npm install
npm run dev:1919
```

如需本地统一预览桌面版和手机版：
- 运行仓库根目录下的统一网关脚本，使用同一个 `1919` 端口承载 `/web-admin/desktop/` 与 `/wx-app/`

### 4. 历史小程序说明
- 仓库仍保留旧小程序代码，便于追溯和比对
- 但手机端正式方向已切换为微信服务号网页 App，不再继续以小程序作为主交付面

## 服务号网页 App 使用说明
- 统一移动入口：`https://你的域名/wx-app/`
- 微信内打开时优先作为服务号工作台使用，桌面设备访问会自动跳转到 PC 后台
- 登录支持账号密码，后续可补充微信网页授权绑定
- 工作台按角色显示待办、下一步节点、流程概览、消息提醒和最近操作
- 申请人重点使用：我的流程、我的资料、材料维护
- 审核者重点使用：待办处理、消息提醒、流程查询和关键节点确认
- 流程页面按 25 步展示，但优先突出当前节点和下一步，已完成步骤折叠到下方
- 材料按步骤和标签归集，不单独拆分总档案夹

## 后台使用说明
- 默认演示/开发管理员账号：`admin / 123456`
- 桌面后台用于完整台账、导出、流程配置和组织管理
- 服务号网页 App 用于微信手机端中的待办处理、消息提醒、资料维护、流程查看和材料上传
- 登录后按角色展示菜单
- 申请人台账支持单位、支部、阶段和关键字筛选
- 流程详情页查看单个申请人的 25 步记录
- 审核审批页处理待审节点
- 统计分析页查看单位/支部维度统计
- 数据导出页下载台账和统计报表
- 流程配置页维护每步起止时间
- 登录页与侧边栏靠后位置提供“更多选项”，可切换“样式1/样式2”
- 工作台首页内置“注意事项”和流程示意，用于强调正式流程要求

## 服务部署说明与要求
- 建议环境：Linux 云主机
- 反向代理：Nginx
- 进程管理：PM2
- 数据库：MySQL 8.x
- 证书要求：后端 API 和后台域名均为 HTTPS
- 上传目录：独立持久化目录，例如 `/data/party-building/uploads`
- 微信绑定：需配置 `WECHAT_APP_ID`、`WECHAT_APP_SECRET`、`WECHAT_SESSION_SECRET`
- 服务号网页授权：需额外配置 `WECHAT_SERVICE_APP_ID`、`WECHAT_SERVICE_APP_SECRET`、`WECHAT_SERVICE_REDIRECT_URI`
- frp 端口规划：
  - `1145 -> 3000`：服务端 API
  - `1919 -> 1919`：后台前端
- Navicat 数据库管理建议继续通过 VMware NAT 或受控内网访问，不通过 frp 暴露数据库
