# 党员发展管理系统

基于微信小程序、Node.js 服务端和 PC 后台的党务管理项目，覆盖发展党员流程、角色权限、流程审核、统计分析和数据导出。

## 项目介绍
- 小程序端：申请人和基层管理角色的移动入口。
- 服务端：统一认证、权限、流程、统计、导出、附件上传。
- PC 后台：查看台账、审核审批、统计分析、流程配置、组织与角色管理。

Gitee 仓库地址：
- [https://gitee.com/Yanhe06/party-building-mini-app.git](https://gitee.com/Yanhe06/party-building-mini-app.git)

## 业务场景
- 发展党员 25 步流程留痕管理
- 申请人基础资料维护
- 支部/单位/组织部多级权限操作
- 注册审核、流程审核、统计下载

## 角色与权限
- 入党申请人：查看与提交本人流程和资料
- 党支部书记：查看本支部、处理支部审核节点、导出本支部数据
- 组织员：查看本单位、审核注册、推进流程、分配角色
- 二级单位党委书记/副书记：查看与审批本单位范围内事项
- 校党委组织部：全校查看、统计、审核、导出
- 超级管理员：组织配置、角色配置、流程配置、全量导出

## 本地开发

### 1. 启动服务端
```bash
cd server
npm install
cp .env.example .env
npm run start
```

说明：
- 服务端默认连接 MySQL
- 启动时会执行 `server/deploy/mysql-init.sql` 并初始化演示数据
- 开发联调配置请参考独立调试文档

### 2. 启动后台
```bash
cd admin-web
npm install
npm run dev
```

可通过 `admin-web/.env.example` 配置 `VITE_API_BASE`。

### 3. 启动小程序
- 使用微信开发者工具打开仓库根目录
- 修改 `utils/config.js` 中的 API 地址
- 将服务端 HTTPS 域名加入小程序合法域名
- 小程序支持显式切换“演示模式”和“服务端模式”

## 小程序使用说明
- 登录页支持账号密码登录
- 登录页支持微信快捷登录
- 首次注册需提交姓名、身份证号、学号/工号和密码
- 工作台展示当前角色、范围和关键指标
- 流程页查看 25 步状态
- 步骤详情页支持上传附件、提交步骤、审核步骤
- 个人资料页支持服务端保存
- 个人资料页支持微信账号绑定与解绑
- 演示场景下可直接使用内置样例数据完成流程展示
- 长按小程序页眉可在“标准版/宣传版”视觉之间隐藏切换

## 后台使用说明
- 登录后按角色展示菜单
- 申请人台账支持单位、支部、阶段和关键字筛选
- 流程详情页查看单个申请人的 25 步记录
- 审核审批页处理待审节点
- 统计分析页查看单位/支部维度统计
- 数据导出页下载台账和统计报表
- 流程配置页维护每步起止时间
- 双击后台标题可在“标准版/宣传版”视觉之间隐藏切换

## 服务部署说明与要求
- 建议环境：Linux 云主机
- 反向代理：Nginx
- 进程管理：PM2
- 数据库：MySQL 8.x
- 证书要求：后端 API 和后台域名均为 HTTPS
- 上传目录：独立持久化目录，例如 `/data/party-building/uploads`
- 微信绑定：需配置 `WECHAT_APP_ID`、`WECHAT_APP_SECRET`、`WECHAT_SESSION_SECRET`
