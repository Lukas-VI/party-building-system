# 党员发展管理系统

基于微信小程序、Node.js 服务端和 PC 后台的党务管理项目，覆盖发展党员流程、角色权限、流程审核、统计分析和数据导出。

## 项目介绍
- 小程序端：申请人和基层管理角色的移动入口。
- 服务端：统一认证、权限、流程、统计、导出、附件上传。
- PC 后台：查看台账、审核审批、统计分析、流程配置、组织与角色管理。

Gitee 仓库地址：
- [https://gitee.com/Yanhe06/party-building-mini-app.git](https://gitee.com/Yanhe06/party-building-mini-app.git)

## 业务场景
- 发展党员 25 步流程留痕管理
- 申请人基础资料维护
- 支部/单位/组织部多级权限操作
- 注册审核、流程审核、统计下载

## 角色与权限
- 入党申请人：查看与提交本人流程和资料
- 党支部书记：查看本支部、处理支部审核节点、导出本支部数据
- 组织员：查看本单位、审核注册、推进流程、分配角色
- 二级单位党委书记/副书记：查看与审批本单位范围内事项
- 校党委组织部：全校查看、统计、审核、导出
- 超级管理员：组织配置、角色配置、流程配置、全量导出

## 本地开发

### 1. 启动服务端
```bash
cd server
npm install
cp .env.example .env
npm run start
```

说明：
- 服务端默认连接 MySQL
- 启动时会执行 `server/deploy/mysql-init.sql` 并初始化演示数据
- 开发联调配置请参考独立调试文档

### 2. 启动后台
```bash
cd admin-web
npm install
npm run dev
```

可通过 `admin-web/.env.example` 配置 `VITE_API_BASE`。
如需按开发服务器联调口径启动后台前端：
```bash
npm run dev:1919
```
如需通过 `https://域名/admin/` 反向代理到后台预览：
- `npm run preview:1919` 会在 `1919` 端口提供 `/admin/` 子路径静态预览服务
- 后台构建资源基址已固定为 `/admin/`

### 3. 启动小程序
- 使用微信开发者工具打开仓库根目录
- 修改 `utils/config.js` 中的 `apiBase` 和 `adminWebUrl`
- 将服务端 HTTPS 域名加入小程序合法域名
- 小程序支持显式切换“演示模式”和“服务端模式”
- 小程序与后台提供两套正式亮色样式，便于在汇报和日常使用间切换

## 小程序使用说明
- 登录页支持账号密码登录
- 登录页支持微信快捷登录
- 首次注册需提交姓名、身份证号、学号/工号和密码
- 工作台展示当前角色、范围和关键指标
- 工作台快捷入口按角色分层展示，不同角色不再共用同一组入口
- 工作台内置“注意事项”和“三阶段二十五步”流程示意，便于按规范办理
- 流程页查看 25 步状态
- 步骤详情页支持上传附件、提交步骤、审核步骤
- 个人资料页按角色分版，申请人、基层管理角色、组织部/管理员显示不同字段
- 个人资料页支持微信账号绑定与解绑
- 演示场景下可直接使用内置样例数据完成流程展示
- 管理角色可通过“小程序 > 工作台 > PC 后台”进入内嵌 `web-view` 后台入口
- `web-view` 仅支持 HTTPS 后台地址，当前默认指向 `https://havensky.cn/admin/`
- 个人资料页底部提供“更多选项”，可切换“样式1/样式2”

## 后台使用说明
- 默认演示/开发管理员账号：`admin / 123456`
- 登录后按角色展示菜单
- 申请人台账支持单位、支部、阶段和关键字筛选
- 流程详情页查看单个申请人的 25 步记录
- 审核审批页处理待审节点
- 统计分析页查看单位/支部维度统计
- 数据导出页下载台账和统计报表
- 流程配置页维护每步起止时间
- 登录页与侧边栏靠后位置提供“更多选项”，可切换“样式1/样式2”
- 工作台首页内置“注意事项”和流程示意，用于强调正式流程要求

## 服务部署说明与要求
- 建议环境：Linux 云主机
- 反向代理：Nginx
- 进程管理：PM2
- 数据库：MySQL 8.x
- 证书要求：后端 API 和后台域名均为 HTTPS
- 上传目录：独立持久化目录，例如 `/data/party-building/uploads`
- 微信绑定：需配置 `WECHAT_APP_ID`、`WECHAT_APP_SECRET`、`WECHAT_SESSION_SECRET`
- frp 端口规划：
  - `1145 -> 3000`：服务端 API
  - `1919 -> 1919`：后台前端
- Navicat 数据库管理建议继续通过 VMware NAT 或受控内网访问，不通过 frp 暴露数据库
